Understanding Agentic AI

What is Agentic AI?

Agentic AI refers to artificial intelligence systems that can autonomously perform tasks, make decisions, and take actions on behalf of users. Unlike traditional AI that simply responds to prompts, agentic AI can:

  • Plan and execute multi-step workflows
  • Use tools and APIs to interact with external systems
  • Make decisions based on context and goals
  • Adapt to changing conditions in real-time
  • Learn from feedback and improve over time

The Goal of Agentic AI

The primary goal is to automate complex workflows and augment human capabilities by:

  • Reducing repetitive tasks: Automating routine work that consumes valuable time
  • Enhancing productivity: Allowing humans to focus on strategic, creative work
  • Improving accuracy: Minimizing human error in data-heavy tasks
  • Enabling scalability: Handling multiple tasks simultaneously without fatigue
  • Bridging systems: Connecting different tools and platforms seamlessly

Installation

1Install Node.js

Download and install Node.js from the official website:

https://nodejs.org/

Verify installation by opening Command Prompt or PowerShell:

node --version
npm --version

2Install Gemini CLI

Open PowerShell or Command Prompt as Administrator and run:

npm install -g @google/generative-ai-cli

Verify installation:

gemini --version

3Get Your API Key

Visit Google AI Studio to generate your API key:

https://makersuite.google.com/app/apikey
Note

Keep your API key secure and never share it publicly.

4Configure API Key

Set your API key as an environment variable:

PowerShell:

$env:GOOGLE_API_KEY="your-api-key-here"

Command Prompt:

set GOOGLE_API_KEY=your-api-key-here

Permanent (System Environment Variable):

  • Open System Properties → Advanced → Environment Variables
  • Click "New" under User variables
  • Variable name: GOOGLE_API_KEY
  • Variable value: your API key
  • Click OK and restart your terminal

MCP (Model Context Protocol) Setup

1Create MCP Configuration Directory

Create a directory for MCP configuration:

mkdir %USERPROFILE%\.gemini
cd %USERPROFILE%\.gemini

2Create MCP Config File

Create a file named mcp_config.json in the .gemini directory:

{
  "mcpServers": {
    "figma": {
      "url": "https://mcp.figma.com/mcp",
      "apiKey": "YOUR_FIGMA_API_KEY"
    },
    "canva": {
      "url": "https://mcp.canva.com/mcp",
      "apiKey": "YOUR_CANVA_API_KEY"
    },
    "notion": {
      "url": "https://mcp.notion.com/mcp",
      "apiKey": "YOUR_NOTION_API_KEY"
    }
  }
}

3Configure Figma MCP

Get your Figma API key:

  • Go to Figma API Settings
  • Click "Generate new token"
  • Copy the token
  • Replace YOUR_FIGMA_API_KEY in the config file
Figma Permissions

Ensure your Figma token has read access to the files you want to interact with.

4Configure Canva MCP

Get your Canva API key:

  • Visit Canva Developers
  • Create a new app or use an existing one
  • Generate an API key
  • Replace YOUR_CANVA_API_KEY in the config file
Canva Setup

You may need to authenticate via OAuth for full access to Canva features.

5Configure Notion MCP

Get your Notion integration token:

  • Go to Notion Integrations
  • Click "New integration"
  • Give it a name and select capabilities
  • Copy the "Internal Integration Token"
  • Replace YOUR_NOTION_API_KEY in the config file

Share pages with your integration:

  • Open the Notion page you want to access
  • Click "Share" → "Invite"
  • Select your integration and grant access

6Configure Chrome MCP

Chrome MCP enables browser automation and web interaction capabilities.

Installation:

npm install -g @modelcontextprotocol/server-chrome

Update your mcp_config.json:

{
  "mcpServers": {
    "figma": {
      "url": "https://mcp.figma.com/mcp",
      "apiKey": "YOUR_FIGMA_API_KEY"
    },
    "canva": {
      "url": "https://mcp.canva.com/mcp",
      "apiKey": "YOUR_CANVA_API_KEY"
    },
    "notion": {
      "url": "https://mcp.notion.com/mcp",
      "apiKey": "YOUR_NOTION_API_KEY"
    },
    "chrome": {
      "command": "mcp-server-chrome",
      "args": ["--headless"]
    }
  }
}

Troubleshooting

Command Not Found

If gemini command is not recognized:

  • Restart your terminal
  • Verify npm global bin path is in your PATH
  • Run: npm config get prefix and ensure it's in your system PATH

API Key Issues

If API authentication fails:

  • Verify your API key is correct
  • Check environment variable is set: echo %GOOGLE_API_KEY%
  • Ensure no extra spaces or quotes in the key

MCP Connection Issues

If MCP servers fail to connect:

  • Verify JSON syntax in mcp_config.json
  • Check API keys for each service are valid
  • Ensure you have granted necessary permissions in each service
  • Check internet connection and firewall settings
